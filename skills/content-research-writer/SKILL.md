# Content Research Writer (PM/BA Edition)

This skill acts as your writing partner for enterprise documentation, helping you transform casual input into formal, professional content while maintaining clarity and consistency.

## When to Use This Skill

- Writing structured enterprise documents (Test Plans, PRDs, BRDs, etc.)
- Formalizing casual notes into professional language
- Getting section-by-section feedback while writing
- Improving clarity and consistency in documentation
- Converting bullet points into full paragraphs
- Ensuring professional tone across documents

## What This Skill Does

1. **Formalization**: Converts casual input into professional documentation language
2. **Section Feedback**: Reviews each section for clarity and consistency
3. **Clarity Enhancement**: Simplifies complex statements, removes ambiguity
4. **Consistency Checking**: Ensures tone and terminology align across sections
5. **Iterative Refinement**: Helps you improve through review loops

## How to Use

### Basic Workflow

1. **Formalize casual input**:
```
Convert this to professional language: "Users can't login sometimes"
```

2. **Get section feedback**:
```
Review this Test Scope section for clarity and completeness.
```

3. **Check consistency**:
```
Does this section match the tone of the previous sections?
```

4. **Refine and polish**:
```
Review the full draft for clarity and professional tone.
```

## Best Practices

### For Formalization
- Preserve the original meaning exactly
- Remove vague language ("maybe", "sometimes", "probably")
- Use specific, professional terminology
- Maintain clarity (avoid over-complication)

### For Feedback
- Be specific: "Is this section too technical?"
- Focus on clarity and completeness
- Check for ambiguity
- Ensure logical flow

### For Consistency
- Track terminology across sections
- Maintain consistent tone
- Use parallel structure in lists
- Align formatting throughout
